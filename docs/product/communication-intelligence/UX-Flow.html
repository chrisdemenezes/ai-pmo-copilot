Princípio — Rev. 3 (Founder)

Refinamento estratégico, incorporado antes da UX Flow: a experiência deixa de ser orientada ao agente e passa a ser orientada ao objetivo do usuário.

O usuário nunca escolhe um modelo de IA. Ele escolhe o que quer entender sobre o projeto. A plataforma decide internamente qual agente executar — isso já é verdade na arquitetura de hoje (o kind escolhido mapeia para um hook/BFF/agente específico, nunca exposto por nome técnico); esta Rev. 3 exige que a linguagem da interface reflita isso com a mesma disciplina.

Isso não é uma decisão isolada desta Feature — é um princípio que se aplica retroativamente às 2 capacidades já reais (Status Executivo, Avaliação de Riscos). A Seção 2 propõe a extensão consistente das 3.

04

Preparando a Evolução Futura (avaliação, sem nova funcionalidade)

O Founder pediu para avaliar, durante esta UX Flow, se já é possível preparar a futura fusão de Project Intelligence, Risk Intelligence e Communication Intelligence numa única experiência — sem construir nada novo agora. Avaliação honesta:

Sim, já é possível — porque o mecanismo já é o mesmo. O Dialog "Analisar Projeto", o ANALYSIS_CATALOG e o Tabs já são, estruturalmente, uma mini-versão da experiência integrada que o Founder descreve: um único ponto de entrada, um catálogo de "coisas que você pode entender sobre o projeto", cada uma mapeada internamente a um agente sem expor esse mapeamento. A fusão futura das 3 Capabilities não vai exigir uma reconstrução deste mecanismo — só crescer o mesmo catálogo. A preparação real não é código novo; é não deixar a linguagem se afastar disso — por isso a Seção 2 propõe alinhar os 3 rótulos agora, enquanto são só 3, em vez de deixar a inconsistência acumular até que a fusão fique mais cara de fazer.

Uma extensão de dado (não de comportamento) fica registrada aqui como recomendação para a Feature Specification, não como decisão tomada: o ANALYSIS_CATALOG poderia ganhar um campo descritivo capability ("project-intelligence" | "risk-intelligence" | "communication-intelligence") — metadado puro, zero mudança de comportamento visível, que serve de documentação e facilita a fusão futura sem forçar nenhuma decisão de UI ainda não aprovada.
