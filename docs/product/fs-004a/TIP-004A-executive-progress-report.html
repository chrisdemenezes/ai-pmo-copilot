Achado real corrigido durante a implementação

Adicionar "Projetos" à navegação quebrou um teste E2E pré-existente do Dashboard (dashboard.spec.ts), que usava getByText("Projetos").first() para localizar o cabeçalho de seção "Projetos" — agora esse texto também existe no rótulo do item de menu (oculto em md/mobile, hidden lg:inline). Corrigido apontando o teste para o role de heading, que desambigua corretamente em qualquer breakpoint. Nenhum comportamento real do produto mudou — o achado é puramente de precisão de seletor de teste, exatamente o tipo de regressão que a suíte automatizada existe para capturar.

Também descoberto: no breakpoint md (rail somente-ícone), o link de navegação não tem nome acessível (rótulo de texto é hidden lg:inline) — os novos testes E2E usam seletor por href em vez de nome acessível para cliques de navegação, funcionando nos 3 breakpoints.
