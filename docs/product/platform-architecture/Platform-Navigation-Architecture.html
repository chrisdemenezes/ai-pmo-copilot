Promover um módulo do Catálogo para a Navegação Visível é sempre um ato deliberado — editar navigation.ts dentro de uma Feature real e aprovada — nunca uma consequência automática de mudar um status no catálogo. Isso é o que impede, estruturalmente, um "menu cenográfico": não existe caminho automatizado do catálogo para a tela do usuário.

2. Regra de entrada na Runtime Navigation

Idêntica à Regra Permanente já registrada na FS-004A — reafirmada aqui, não reinventada. Um módulo só entra em navigation.ts quando possuir, todos ao mesmo tempo:

  • rota real
  • experiência funcional
  • dados reais ou derivados
  • estados de loading, vazio e erro
  • testes
  • aderência à Baseline Visual

NavItem permanece exatamente como definido na FS-002 — {"{"}label, href, icon{"}"}, sem campo id, sem status, sem metadado de ciclo de vida. A primeira versão deste documento havia proposto reintroduzir um campo id para o registro de módulos; essa proposta caduca aqui — a Runtime Navigation não precisa de referência cruzada nem de landing page, então o motivo que justificaria o campo não existe no runtime. Ele poderia voltar a fazer sentido dentro do Catálogo (documentação), nunca em navigation.ts.

3. Módulos contextuais — o caso Workspace

Workspace é real (FS-004) mas exige project_name — não tem uma rota utilizável sem esse parâmetro, então nunca recebe item próprio na Runtime Navigation. É alcançado a partir de Projetos e, quando aplicável, do Dashboard — exatamente como já funciona hoje. Nenhuma mudança sobre isso.

4. Organização progressiva dos grupos

Os 5 grupos (Core, Intelligence, Operations, Analytics, Administration) organizam o Catálogo, para planejamento. A Sidebar não precisa de cabeçalhos de grupo enquanto a Runtime Navigation tiver poucos itens — hoje 1, em breve 2. Agrupamento visual na Sidebar é decisão adiada até existir volume real de itens visíveis que justifique — não implementado, não desenhado agora.

5. Como novos módulos entram sem refatorar a Sidebar

Sidebar já mapeia NAV_ITEMS genericamente (FS-002) — adicionar um módulo real é sempre uma linha nova no array, nunca uma mudança de componente. Confirmado, não modificado, por esta arquitetura.

export const NAV_ITEMS: NavItem[] = [
  { label: "Dashboard", href: "/dashboard", icon: LayoutDashboard },
  // + "Projetos" quando a FS-004A for implementada -- mesma forma, uma linha
];
