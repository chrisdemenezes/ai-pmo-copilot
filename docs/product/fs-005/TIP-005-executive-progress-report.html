Achado real corrigido durante a implementação

O mock de E2E (mock-backend.mjs) é um processo único, compartilhado por todos os arquivos de spec e pelos 3 breakpoints dentro de uma mesma execução do Playwright. A nova rota POST /api/projects/analyze muta o estado fictício (SAMPLE, WORKSPACE_SUMMARY, ANALYSES) em memória para provar a jornada — mas sem reset, essa mutação vazava para testes de outros arquivos de spec (Dashboard, Projetos) que corriam depois, na mesma execução. Corrigido com um endpoint __control/reset-fixtures que restaura snapshots imutáveis capturados na inicialização do mock, chamado no beforeEach dos 3 arquivos de spec que dependem desses dados — não só do arquivo que introduziu a mutação. Sem esse achado, a suíte teria ficado dependente da ordem de execução dos testes, um risco de flakiness silencioso.
