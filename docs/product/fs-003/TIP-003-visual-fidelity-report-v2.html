4. O que permanece fora, por decisão

Dependente de funcionalidade futura

  • Painel "Risk Radar" ao lado da tabela — exige nova rota de API (Release Plan 0.3, Épico 2)
  • Grid de 2 colunas (tabela + Risk Radar) — só faz sentido quando o Risk Radar existir
  • Link "Ver todos" nos cabeçalhos — não existe uma segunda view/rota para linkar
  • Sparkline e indicador de tendência nos KPIs — exigem série histórica que o produto não persiste

Fora do escopo

  • Barra de progresso fictícia nos KPIs (denominador que não existe para os 3 KPIs reais)
  • KPIs "Budget Utilizado", "Sprint Velocity", "Capacidade Time" — campos inexistentes no schema
  • Colunas de tabela "Progresso", "Cronograma", "PM" (com avatar) — dados não rastreados
  • Subtítulo "Q3 2025" / "projetos ativos" — produto não tem conceito de trimestre ou status "ativo"
  • Qualquer conteúdo novo inventado para os estados vazio/erro/carregamento — o Figma não os modela
