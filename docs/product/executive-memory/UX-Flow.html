01

Princípio: lembranças, não informações

Diretriz do Founder, elevada a princípio-guia: "Executive Memory não apresenta informações. Executive Memory apresenta lembranças."

A diferença não é estilística -- é funcional. Informação é um dado que o executivo ainda não tem e precisa avaliar (ex.: "Status: Crítico", já coberto pelo Executive Brief). Lembrança é um fato que o executivo já viveu antes, e que a plataforma devolve no momento certo, sem ele pedir -- nunca um dado novo a processar, sempre um reconhecimento de algo que já era real.

Corolários concretos, válidos para toda decisão de design desta UX Flow:

  • Uma lembrança nunca substitui ou duplica um campo já existente do card (Status, contexto, próximo movimento) -- ela só acrescenta, nunca compete (Decision Context Preservation, Journey §07).
  • Uma lembrança é sempre sobre o mesmo objeto que o executivo já está olhando -- nunca uma lembrança de outro projeto ou de um padrão do portfólio (fronteira com Organizational Intelligence, Journey §2).
  • Uma lembrança nunca é redigida como se fosse a primeira vez que aquele fato existe -- ela é sempre enquadrada como algo que retorna ("mudou", "persiste", "reapareceu"), nunca como uma descoberta nova.
02

Os 3 Memory Signals

Únicos 3 tipos autorizados nesta V1 -- cada um mapeado 1:1 a um cálculo já aprovado na User Journey §6, redigido sempre como o menor fato necessário.

Memory SignalO que significaRedação (menor fato necessário)Fonte real
Mudou O valor observável mudou entre as 2 análises mais recentes do mesmo tipo Mudou: Atenção → Crítico list_analyses(kind="status", limit=2) -- health_status anterior vs. atual
Persistiu O mesmo estado de atenção real (crítico ou em atenção) se repete por 2+ análises seguidas -- nunca aplicado a um estado saudável, que não exige memória (Journey §4) Persiste em Crítico (3ª análise seguida) Contagem de análises consecutivas de status com o mesmo health_status em zona de atenção
Reapareceu Um risco equivalente já havia sido identificado em uma análise anterior do mesmo projeto Reapareceu (3ª vez) list_analyses(kind="risk") do projeto -- contagem real de ocorrências equivalentes
Disciplina de redação (Decision Context Preservation, Executive Trust): cada Memory Signal é uma frase curta com um fato verbatim -- nunca uma narrativa ("este projeto tem se deteriorado consistentemente..."), nunca um relatório (uma lista de todas as ocorrências), nunca um histórico completo. Só o nome do signal, o dado real (rótulo de status ou contagem), e nada além disso. A redação exata final (pontuação, capitalização) é decisão da Feature Specification -- o formato e a disciplina ficam fixados aqui.
03

Composição: discreta, contextual, nunca competitiva

Avaliação explícita exigida pelo Founder.

  • Discreta: um Memory Signal é um chip/tag pequeno, tom neutro (reaproveitando a mesma disciplina visual já usada na camada "sem sinal de atenção" da Executive Portfolio View) -- nunca a cor de alerta do Status/Risco, mesmo quando o signal é sobre um estado crítico. A cor de atenção já pertence ao dado principal do card; a lembrança não pode competir por ela.
  • Contextual: aparece sempre ao lado do campo real a que se refere -- "Mudou" perto do Status; "Reapareceu" perto do risco específico -- nunca isolado no topo ou rodapé do card, como se fosse um destaque próprio.
  • Nunca competitiva: menor, mais discreto que qualquer campo principal do card (Status, contexto, próximo movimento) -- nunca em negrito, nunca maior que o texto ao redor, nunca antes do conteúdo principal na ordem de leitura.

Componente candidato: o mesmo chip/badge já usado em outras Capabilities (ex.: os chips de status das Architecture Reviews) -- nenhum componente visual novo, mesma disciplina de Executive Component Library já aplicada em Decision Center e Portfolio Intelligence.

04

Silêncio confirmado

Confirmação direta ao Founder: Executive Memory permanece silenciosa quando não agrega valor -- mesma regra já fixada na User Journey §4, agora nomeada por signal.

Memory SignalFica em silêncio quando
MudouPrimeira análise daquele tipo (sem "antes" real) -- ou o valor não mudou entre as 2 mais recentes
PersistiuMenos de 2 análises seguidas no mesmo estado de atenção -- ou o estado repetido é saudável
ReapareceuO risco aparece pela 1ª vez (contagem = 1)

Nenhum dos 3 silêncios é um erro ou um estado vazio a preencher -- é, ele mesmo, a resposta honesta ("nada digno de lembrança aqui"), mesma lógica do Princípio da Atenção já aplicada em toda a plataforma.

05

Sem estados próprios, sem navegação adicional

Confirmação direta ao Founder.

  • Nunca cria estados próprios de carregamento: um Memory Signal resolve junto com o dado que o card já busca (mesma requisição/hook, nunca uma chamada adicional visível ao executivo) -- se ainda não resolveu, o card simplesmente não mostra o signal ainda, nunca um Skeleton dedicado a ele.
  • Nunca exige navegação adicional: nenhum Memory Signal é clicável, nenhum leva a uma tela própria -- ele é texto de apoio dentro de um card que já é, ou não, clicável por conta própria (Executive Decision Card, Executive Portfolio Card).
06

Nunca interrompe o Executive Loop

Confirmação direta ao Founder.

O Executive Loop (Pergunta → Compreensão → Decisão → Ação → Retorno) continua exatamente como está em cada Capability -- Executive Memory nunca insere um passo novo nele. Um Memory Signal é lido no mesmo momento em que o resto do card já é lido (Compreensão ou Decisão, conforme a superfície), nunca exige uma pausa, uma confirmação ou uma decisão própria antes de prosseguir. Se todos os 3 signals estiverem em silêncio (Seção 4), o Loop se comporta exatamente como se Executive Memory não existisse.

09

Fora de Escopo (confirmado)

  • Qualquer 4º tipo de Memory Signal -- só Mudou, Persistiu, Reapareceu nesta V1.
  • Qualquer narrativa, relatório ou histórico completo dentro de um signal -- sempre o menor fato necessário.
  • Qualquer tela dedicada de memória, item de navegação novo, ou estado de carregamento próprio.
  • Qualquer padrão agregado do portfólio -- pertence a uma futura Organizational Intelligence.
  • Qualquer refatoração do Executive Brief, Executive Decision Card ou Executive Portfolio Card além do signal em si.
  • Qualquer configuração pelo usuário de quais signals aparecem -- a regra de silêncio é sempre fixa.
