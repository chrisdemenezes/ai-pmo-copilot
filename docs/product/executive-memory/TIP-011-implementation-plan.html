01

Entrega de Valor

1. Em qual momento o usuário começa a perceber valor?

Assim que "Mudou" e "Persistiu" existirem no Executive Brief -- não é preciso esperar "Reapareceu" nem a regra de seleção completa. É o menor incremento que já responde, com dado real, à pergunta central da Discovery: "o que o executivo precisa lembrar antes de decidir?".

2. Qual será a primeira funcionalidade utilizável?

O Executive Brief já mostrando "Mudou: X → Y" ou "Persiste em X (Nª análise seguida)" quando aplicável -- funciona sozinho, de ponta a ponta (dado já existente → buildStatusInsight → chip), sem depender do sinal de risco.

3. Como organizar para entregar valor incremental mantendo a arquitetura completa?

2 incrementos (Seção 4), cada um demonstrável sozinho -- mesma disciplina de toda Capability anterior. Ao final do Incremento 2, o escopo integral da FS-010 está entregue, incluindo a One Memory Insight Rule aplicada aos 3 signals.

4. Qual é o caminho mais curto entre abrir a STRATECH e entender um projeto com todo o contexto necessário?

Hoje: para saber se um status já mudou ou um risco já reapareceu, o executivo abre o Histórico completo e compara manualmente. Depois desta Feature: o mesmo Executive Brief que ele já lê hoje já chega com esse fato, sem nenhum passo adicional.

02

Redução do esforço de reconstrução de contexto

TTD e Return Rate (RR) já são princípios oficiais (TIP-008 §08) -- esta seção os aplica a Executive Memory, cujo efeito é mais qualitativo que os das Capabilities anteriores: ela não organiza um elo do Executive Loop, ela reduz o esforço de compreensão dentro do elo "Compreensão" já existente (Architecture Review §2/§6).

Efeito esperado: nos casos em que um Insight real existe, elimina o custo de abrir o Histórico completo e comparar manualmente. Nos casos de silêncio (a maioria, por desenho), nenhum efeito, nunca um custo adicional (Architecture Review §6, pergunta 5). Nenhuma métrica implementada, nenhuma telemetria criada nesta entrega.
03

Princípios Preservados

  • Executive Behavior ("Lembrar") -- FS-010 §02; nenhuma tarefa introduz uma ação executada dentro do próprio Insight.
  • One Memory Insight Rule -- Architecture Review §3.3; selectPrimaryInsight() é a única implementação da regra de seleção, nunca duplicada por superfície.
  • Platform Integrity Assessment -- Architecture Review §6; nenhuma tarefa introduz competição de responsabilidade com outra Capability.
  • Executive Trust -- nenhuma tarefa produz texto fabricado, narrativa ou interpretação.
  • Progressive Context / Progressive Purpose -- toda tarefa aprofunda só o Executive Brief, nunca repete Dashboard nem antecipa outra Capability.
  • Decision Context Preservation -- cada Insight permanece o menor fato necessário (um delta ou uma contagem).
  • Executive Responsibility Separation -- nenhuma tarefa introduz uma pergunta executiva já respondida por outra Capability.
  • Arquitetura evolutiva / máxima reutilização -- toda tarefa reaproveita uma rota/hook/composição já real; zero rota nova, zero serviço novo (escopo decidido na Architecture Review §4).
04

Incrementos de Valor ao Executivo

Cada incremento responde as 3 perguntas obrigatórias (TIP-009 §08), além de continuar demonstrável, gerador de valor real e independente da conclusão da Capability inteira.

Incremento 1 -- demonstrável isoladamente

Executive Memory Insight de Status: Mudou e Persistiu

Qual valor entrega

Primeira vez que o Executive Brief lembra o executivo de uma mudança ou de uma estagnação real de status, sem ele precisar reconstruir isso manualmente no Histórico completo.

Qual comportamento executivo habilita

Lembrar (FS-010 §02) -- primeira aplicação concreta, sobre o sinal de Status.

Qual parte da plataforma passa a reutilizar

Nenhuma ainda -- nascimento do módulo (memory-insights.ts), consumindo dado que o Executive Brief já busca, sem nenhum consumidor além dele.

Incremento 2 -- reaproveita 100% do Incremento 1

Executive Memory Insight de Risco: Reapareceu, e a One Memory Insight Rule completa

Qual valor entrega

Cobertura dos 3 Memory Signals, com a garantia arquitetural de que o executivo nunca vê mais de um Insight ao mesmo tempo, mesmo quando mais de um seria tecnicamente verdadeiro.

Qual comportamento executivo habilita

Lembrar, agora completo -- nenhuma lacuna entre os 3 Memory Signals aprovados na UX Flow.

Qual parte da plataforma passa a reutilizar

selectPrimaryInsight() nasce como a única implementação da hierarquia de seleção (Architecture Review §3.3) -- reutilizável por qualquer superfície futura que também precise escolher entre candidatos de memória.
