Release Roadmap 0.1 → 0.5

Detalhamento de objetivo, escopo, fora de escopo, dependências, riscos, critérios de aceite e evidência esperada para cada release. Sequência inicial, não limitação da visão futura (Blueprint Seção 20).

Origem Blueprint v2.0, Seção 20Data 2026-07-16

Release 0.1 — Enterprise Foundation

Objetivo
Fundação de identidade, organização e projeto real -- pré-requisito de tudo mais.
Escopo
Organização (1 por instalação), autenticação individual, hash seguro de senha, sessão individual, RBAC com 4 papéis (Organization Admin, PMO, Project Manager, Viewer), Projeto como entidade persistente, vínculo usuário↔projeto, migração de analysis_records, auditoria básica, telas administrativas mínimas.
Fora de escopo
Portfólio, Programa, convite de usuário além do básico, atores externos, SSO, qualquer Acelerador de IA novo, Integration Hub, Orquestração.
Dependências
Nenhuma -- é a primeira release.
Riscos
Migração de analysis_records pode gerar projetos duplicados (mitigado por ADR-V2-003); mudança de banco para schema relacional real exige Alembic desde o primeiro commit.
Critérios de aceite
Os 12 listados no direcionamento do Founder: 2 usuários autenticados, 2 perfis com permissão distinta, 2 organizações sem acesso cruzado, criação real de projeto, associação usuário↔projeto, restrição por papel, análise de IA vinculada ao projeto real, dashboard com entidades reais, histórico de auditoria, instalação limpa, testes automatizados, documentação atualizada.
Evidência esperada
Testes de autorização e segregação entre organizações, testes de migração (contagem 1:1), captura de tela do fluxo completo (criar org → convidar usuário → criar projeto → associar → analisar → ver dashboard).

Release 0.2 — Portfolio & Governance Foundation

Objetivo
Completar a hierarquia (Portfólio, Programa) e abrir a plataforma a atores além do usuário interno.
Escopo
Portfólio, Programa, equipes, convite de usuário, stakeholders, patrocinadores, atores externos (cliente/fornecedor/consultor/auditor/parceiro), expansão do RBAC para o conjunto completo de 14 papéis (ou o subconjunto que se mostrar necessário, com justificativa).
Fora de escopo
Aceleradores de IA novos, Integration Hub, Orquestração.
Dependências
Release 0.1 (Organização, Projeto, RBAC básico).
Riscos
Expansão de RBAC de 4 para até 14 papéis pode revelar lacunas no motor de permissões não visíveis com o conjunto reduzido.
Critérios de aceite
Portfólio/Programa como entidades reais relacionadas a Projeto; ator externo acessando apenas o escopo convidado, nunca herdando permissão de organização.
Evidência esperada
Testes de segregação para ator externo; teste de escopo por portfólio/programa/projeto.

Release 0.3 — AI Foundation

Objetivo
Primeiro framework estruturado de Aceleradores operando sobre entidades reais.
Escopo
Contrato de Acelerador (Seção 11 do Blueprint) implementado; Project Health, Risk Intelligence e Meeting Intelligence migrados dos agentes do RC-1 para operar sobre a entidade Projeto real; evidência, confiança, validação humana e auditoria de análise.
Fora de escopo
Integration Hub, Orquestração, os demais 9 Aceleradores do mapa.
Dependências
Release 0.1 (Projeto real); Release 0.2 opcional (não bloqueia, mas enriquece o contexto).
Riscos
Migrar os agentes do RC-1 sem quebrar as Capabilities existentes que dependem deles -- exige testes de regressão completos antes e depois.
Critérios de aceite
Toda análise de IA carrega evidência e confiança; nenhuma recomendação com ação crítica é executada sem validação humana.
Evidência esperada
Suíte de regressão completa do RC-1 permanece verde; novos testes de auditoria de invocação de Acelerador.

Release 0.4 — Integration Hub

Objetivo
Validar o padrão arquitetural de conector com uma integração real.
Escopo
Arquitetura de conectores (Seção 12), credenciais por organização, mapeamento de entidades, sincronização, monitoramento, tratamento de erros, 1 conector de referência (avaliação técnica define qual -- ClickUp, Teams, ou importação estruturada de MS Project são as opções iniciais sugeridas).
Fora de escopo
Qualquer outro conector; escrita de retorno, a menos que o conector escolhido a suporte com baixo risco.
Dependências
Release 0.1 (credenciais por organização), Release 0.2 (Projeto/Portfólio como alvo de sincronização).
Riscos
Autenticação segura com um sistema terceiro real introduz superfície de ataque nova -- exige revisão de segurança dedicada antes de ir a produção.
Critérios de aceite
Sincronização idempotente demonstrada (reprocessar o mesmo evento não duplica dado); erro tratado sem falha silenciosa; origem do dado sincronizado rastreável.
Evidência esperada
Teste de idempotência, teste de reprocessamento após falha simulada.

Release 0.5 — Event Orchestration

Objetivo
Coordenar fluxos entre domínios e sistemas externos com governança.
Escopo
Event Bus, taxonomia de eventos (Event Map), regras e workflows simples, human-in-the-loop obrigatório para ação crítica, auditoria e observabilidade de orquestração.
Fora de escopo
Workflows totalmente configuráveis por organização (evolução pós-0.5); automação plena sem aprovação para qualquer ação com efeito em sistema externo.
Dependências
Releases 0.1-0.4 (todos os domínios produtores de evento já precisam existir).
Riscos
Maior complexidade de depuração em fluxos assíncronos multi-sistema -- exige observabilidade de ponta a ponta desde o primeiro workflow implementado.
Critérios de aceite
Fluxo de referência completo demonstrado (ex.: reunião → decisão/tarefa extraída → validação humana → sincronização com sistema externo → indicador recalculado → notificação), com toda etapa auditável e reversível em caso de falha.
Evidência esperada
Teste E2E do fluxo de referência completo; teste de compensação/rollback em falha simulada de uma etapa intermediária.
