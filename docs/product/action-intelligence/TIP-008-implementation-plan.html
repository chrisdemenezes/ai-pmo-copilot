01

Entrega de Valor

1. Em qual momento o usuário começa a perceber valor?

Assim que a seção "Ações" existir dentro do Workspace de um projeto já real -- não é preciso esperar a página de portfólio, o item de navegação novo, ou as linhas de contexto nos 3 Briefs. É o menor incremento que já responde, para um projeto, as 3 perguntas fixas da FS-007 §01.

2. Qual será a primeira funcionalidade utilizável?

A seção "Ações" do Workspace, com os itens de reunião já reais daquele projeto, agrupados por urgência. Funciona sozinha, de ponta a ponta (backend → BFF → hook → action-momentum.ts → componente), sem depender de nenhuma outra fatia desta Feature.

3. Como organizar para entregar valor incremental mantendo a arquitetura completa?

3 fatias (Seção 4), cada uma shippable e demonstrável sozinha, nenhuma delas exigindo retrabalho da anterior -- a lógica de dado (serviço, rota, hook, action-momentum.ts) é construída uma única vez no Incremento 1 e só reaproveitada, nunca duplicada ou reescrita, nos Incrementos 2 e 3. Ao final do Incremento 3, o escopo integral da FS-007 está entregue -- os incrementos são uma ordem de construção orientada a valor, não um corte de escopo.

4. Qual é o caminho mais curto entre abrir a STRATECH e tomar uma decisão executiva?

Hoje: Login → Dashboard → Projetos → escolher um projeto → Workspace → ler o Brief inteiro. Depois desta Feature (Incremento 3): Login → Dashboard → clicar no KPI "Ações pendentes" → tela "Ações" já com o item mais urgente do portfólio inteiro destacado no topo. Primeira vez que uma decisão executiva não exige escolher um projeto antes.

02

Time To Decision (TTD) -- registro conceitual

Novo princípio de produto, registrado apenas conceitualmente -- nenhuma métrica implementada, nenhuma telemetria criada nesta Feature ou em qualquer código desta entrega.

Definição: tempo necessário para que um executivo abra a STRATECH, compreenda sua situação, e consiga tomar uma decisão.

Por que registrar agora: a resposta à pergunta 4 (Seção 1) é, na prática, uma descrição qualitativa do TTD -- esta Feature é a primeira a encurtar esse caminho de forma mensurável no futuro (menos passos de navegação até uma decisão). Registrar o princípio agora, sem instrumentá-lo, significa que decisões futuras de UX podem ser avaliadas contra ele explicitamente, mesmo antes de existir uma métrica real. Instrumentação real (telemetria, dashboards de produto) permanece uma evolução futura, fora do escopo de qualquer Capability atual -- não faz parte de Enterprise Essentials nem de Enterprise Expansion no STRATECH V1 Completion Board; seria registrado ali quando priorizado.

Dashboard + 3 Executive Briefs

KPI "Ações pendentes" vira link (Incremento 2 já existe para receber o clique). Nenhum componente dos Incrementos 1-2 é alterado -- só consumido em mais lugares.

08

Princípios Permanentes Registrados

Três princípios oficiais da STRATECH, formalizados pelo Founder nesta etapa. Nenhum altera o escopo deste TIP -- passam a orientar a evolução futura da plataforma.

Incrementos de Valor

Substitui "Fatias Técnicas" (já renomeado nas Seções 1 e 4 deste documento) como conceito permanente: toda implementação futura é organizada em incrementos que precisam, cada um, ser demonstráveis, gerar valor real, ser potencialmente entregáveis, e nunca depender da conclusão da Capability inteira para mostrar resultado. Este TIP já nasceu seguindo essa forma -- a mudança é de nome e de status (de convenção local para princípio oficial), não de conteúdo.

Executive Loop

Novo conceito, inaugurado por esta Capability: toda Capability deve ser analisada como um ciclo Pergunta → Compreensão → Decisão → Ação → Retorno natural à plataforma. Refina o ciclo Compreender → Decidir → Agir → Acompanhar → Volta já usado na Architecture Review (§03) -- começando explicitamente pela Pergunta executiva (Executive Questions Framework, pergunta 1), não pela Compreensão. Registrado apenas como princípio nesta etapa; consolidação num documento estratégico próprio fica para quando o Founder priorizar.

Executive Decision Performance

Time To Decision (TTD) permanece princípio oficial (registrado na Seção 2). Novo KPI conceitual acrescentado: Return Rate (RR) -- capacidade da STRATECH de induzir o retorno natural do executivo à plataforma para continuidade do ciclo decisório. Mesmo tratamento do TTD: nenhuma métrica implementada, nenhuma telemetria criada, só o conceito registrado para orientar decisões futuras de UX (ex.: a Arquitetura Review §03 já avaliou, sem instrumentação, se o desenho de Action Intelligence favorece esse retorno).
