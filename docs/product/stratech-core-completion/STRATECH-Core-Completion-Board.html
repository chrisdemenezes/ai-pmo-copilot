4. Caminho crítico do V1

Por decisão do Founder, o caminho crítico agora é um só: fechar o ciclo Compreender → Decidir → Agir → Acompanhar através do Action Intelligence. Executive Brief Contract, Decision Catalog/Domain/ Context e todo o Hardening (Enterprise Essentials) continuam corretos e pertencem ao V1, mas foram explicitamente adiados -- não competem pelo caminho crítico desta fase.

5. Sequência de implementação (reorganizada)

  1. Agora: Discovery → User Journey → UX Flow → Feature Specification → Architecture Review de Action Intelligence (Product Core). Nenhuma implementação até aprovação do Founder no TIP.
  2. Depois do Action Intelligence: Executive Brief Contract → Decision Catalog/Domain/Context (mesma ordem da Rev. 1 -- só a posição na fila muda, não a lógica interna).
  3. Em paralelo a qualquer um dos anteriores, quando autorizado: Enterprise Essentials (gate de E2E, versionamento de API, model_version, backup/restore) -- infraestrutura pura, não compete por superfície de código com nenhum dos itens acima.
  4. Fora desta fase: Enterprise Expansion (V2) -- não inicia antes do V1 estar concluído.
Por instrução explícita do Founder, o item 3 (Enterprise Essentials) também não inicia agora, apesar de poder rodar em paralelo sem conflito técnico -- toda a capacidade de execução desta fase está dedicada ao Action Intelligence.
