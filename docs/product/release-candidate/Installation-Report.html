STRATECH V1 RC-1 — Installation Report

Relato da preparação e validação de um ambiente de execução local completo para a STRATECH V1 RC-1, incluindo uma instalação limpa real simulando uma máquina sem nenhum ambiente pré-existente. Nenhum desenvolvimento funcional foi iniciado -- apenas documentação, automação de setup e hardening, categorias explicitamente permitidas durante o RC-1.

Commit validado d9f8512 Data 2026-07-16 Resultado Instalação limpa sem impedimentos técnicos

Método

Para simular genuinamente "uma máquina sem ambiente previamente preparado" sem arriscar o ambiente de trabalho desta sessão, um clone isolado do repositório (branch claude/stratech-permanent-principles-yjnm74, commit d9f8512) foi feito em um diretório à parte, sem .venv, sem web/node_modules e sem demo/.env -- as três únicas formas de "estado de ambiente já preparado" que existem neste repositório. A partir daí, todo o procedimento de instalação foi executado do zero, exatamente como um novo desenvolvedor (ou o Founder) faria.

Procedimento executado

git clone --branch claude/stratech-permanent-principles-yjnm74 <repo> clean-test
cd clean-test
bash scripts/rc1-local-start.sh

Resultado: verificação de Python 3.11.15 e Node.js v22.22.2 (ambos presentes e compatíveis), criação do .venv, instalação de requirements.txt, instalação de 513 pacotes npm em web/, criação de demo/.env com SESSION_SECRET gerada, início do backend (saudável em menos de 5s) e do frontend (saudável em menos de 30s) -- sem nenhuma intervenção manual.

Achados e correções (Tarefa 1 -- revisão da documentação de instalação)

A revisão da documentação de instalação encontrou uma lacuna real: nem README.md (raiz) nem web/README.md descreviam como subir a plataforma completa (backend + frontend) -- ambos cobriam apenas o backend isoladamente, via curl direto na API, sem nunca mencionar o frontend, o Demo Mode, ou demo/start-demo.sh. Um novo desenvolvedor seguindo "exclusivamente a documentação" não conseguiria, a partir do README.md raiz sozinho, chegar a uma STRATECH V1 rodável.

Limitação honesta desta validação: os scripts .bat/.ps1 foram escritos traduzindo fielmente a lógica já validada em scripts/rc1-local-start.sh (mesmas variáveis, mesma sequência, mesmo fallback de portas), mas não foram executados nesta revisão -- o ambiente desta sessão é Linux, sem Windows disponível para validar cmd.exe/PowerShell de fato. Recomenda-se uma validação real em uma máquina Windows antes do primeiro piloto que dependa deles.

Conclusão

A STRATECH V1 RC-1 pode ser instalada e executada localmente do zero com um único comando (macOS/Linux) ou dois comandos (Windows), sem impedimento técnico real encontrado. O único problema genuíno identificado nesta revisão foi de documentação (READMEs desatualizados), já corrigido. Ver Founder-Quick-Start.html para os comandos exatos.
