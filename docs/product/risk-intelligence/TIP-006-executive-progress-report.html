Achado real durante a verificação (achado de produção, não hipotético)

Verificação contra o backend real (Demo Mode) encontrou um crash genuíno: parse_structured_output marca structured: true só quando o texto da IA é JSON válido — nunca garante que o JSON tem os campos do schema esperado. O MockLLMProvider do Demo Mode reproduz o último response_file escrito, independente de qual agente foi chamado; uma chamada real de Avaliação de Riscos recebeu de volta um JSON no formato de Status Executivo e quebrou em data.model_output.risks.filter(...). Corrigido com hasRiskShape/hasStatusShape (web/lib/workspace/types.ts) — guardas de runtime aplicadas em todo lugar que lê model_output, incluindo o drill-down de Histórico (analysis-history.tsx), que tinha a mesma lacuna latente desde o TIP-004. Verificado de novo: sem crash, fallback honesto ("Resposta da IA não estruturada nesta análise").
